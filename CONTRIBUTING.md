# Contributing to garrison-plugin-minecraft

Contributions are welcome! Here's how to get started.

## How to contribute

1. **Fork** the repository on GitHub
2. **Clone** your fork locally
3. **Create a branch** for your changes (`git checkout -b my-feature`)
4. **Make your changes** and test them
5. **Commit** with a clear message describing the change
6. **Push** your branch to your fork
7. **Open a Pull Request** against the `main` branch

## Guidelines

- Follow the existing code style and patterns
- Keep changes focused — one feature or fix per PR
- Test your changes against a running Minecraft Java Edition server with RCON enabled
- Update `schema.py` if adding new command definitions
- Update `README.md` if adding user-facing features

## Reporting issues

Open an issue on GitHub with:

- Minecraft server version
- Steps to reproduce
- Expected vs actual behavior
- Relevant RCON responses or error messages
